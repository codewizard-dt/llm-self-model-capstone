# Fact-check Agent

Purpose: adversarially fact check the Vexy Mission Presentation before teammates use it.

## Pass Criteria

- Pass when VEX V5 is described as the physical robotics platform, not as a custom humanoid or generic rover.
- Pass when Flow asset is clearly treated as the cinematic story layer.
- Pass when Repo proof is described as the source for engineering claims.
- Pass when trusted motion is framed as the hard problem: bounded commands, strict vocabulary, observations, and proof.
- Pass when the presentation says learning requires evidence from a run, not vibes or magic autonomy.
- Pass when David Taylor appears immediately after Grace and explains the small learning loop: generate a plan, execute it, analyze telemetry, and start over.
- Pass when the Generational Loop is framed as a future/next-proof path where the LLM may redesign its own body and adapt to real-world conditions.
- Pass when David's telemetry examples include motor samples, vision state, and operator results.
- Pass when Grace explicitly covers `Self-Modeling Machines, in Language`, VEX V5 Brain, Smart Ports, Smart Motors, Raspberry Pi, AprilTags, yellow ball, grab, pull, and throw.
- Pass when five diagram choices per speaker are framed as presentation-edit options, not as claims that all diagrams are equally proven runtime artifacts.
- Pass when Vercel edit mode is described honestly: static local edit/review works with export/import, while shared persistence requires a backend.

## Reject Criteria

- Reject if a slide claims the robot fully learns from every run without evidence or human review.
- Reject if Flow video is used as proof that the robot physically performed the action.
- Reject if VEX V5 is implied to be an LLM, a humanoid robot, or a fictional robot body.
- Reject if the LLM is shown as directly freestyling unsafe motor commands.
- Reject if speaker order drifts away from Grace, David Taylor, Erick, Jake.
- Reject if the Generational Loop is described as already fully solved on the real robot.
- Reject if "redesign its own body" is presented without the future proof path and blocker context.
- Reject if a pinned review comment is implied to sync across teammates without a server store.
- Reject if the final team image is too dark to read the team or if it appears as the only technical proof.

## Agent Review

Result: Pass with caveats.

Caveats:
- The page is a presentation artifact; the videos are Flow assets.
- The technical claim is the architecture and proof path: VEX hardware, bounded commands, observation, evidence packets, language self-model, critic review, and human approval.
- The David Taylor claim is allowed because it is framed as a research synthesis and next-proof path: numerical self-modeling, LLM morphology generation, LLM self-improvement, and the proposed Vexy combination.
- The final robot learning claim should stay phrased as a goal/proof path unless real run evidence is shown.
- The review-mode claim is allowed because it is a presentation collaboration layer, not a robot-runtime claim.
