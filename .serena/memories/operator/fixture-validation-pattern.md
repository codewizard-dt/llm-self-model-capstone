Operator fixture pairs live under `operator/fixtures` as `contract_*.json` plus `outline_*.json`. The ROS operator validates contract fixtures with `vexy_ros.operator.core.validate_contract_line_shape`, which requires `schema_version`, `session_id`, `generation`, `round`, `task`, non-empty `motor_samples`, `predicted`, and `gap`. Outline fixtures are JSON arrays of operator method-call tuples and should be validated with `parse_operator_method_plan`; supported calls include `locate_nearest_apriltag`, `orient_to_tag(tag_index)`, `move_to_tag(tag_index, target_distance_m=...)`, `grab`, `lift`, and `release`. For 12-inch AprilTag standoff fixtures, use `target_distance_m: 0.3048`.
