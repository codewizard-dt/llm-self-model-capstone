# Research: 3D Printer File Formats

> 3D printing requires two distinct file types: a **model file** (STL, OBJ, 3MF, STEP) that describes the 3D shape, and a **machine instruction file** (G-code) that the printer actually executes. The model file goes through a slicer application — software that converts geometry into layer-by-layer printer commands. STL is the universal entry-point format today; 3MF is the emerging modern standard. Every printer ultimately runs G-code (or a proprietary variant), never the model file directly.

## Research Questions

1. What file formats do 3D printers accept as input?
2. What is the most common/universal 3D printing file format and why?
3. What is the slicing workflow and which file formats are produced at each stage?
4. What formats do CAD tools export for 3D printing?
5. Are there differences between consumer FDM and professional/industrial printer formats?

## Key Findings

### The Three-Stage Workflow

Every 3D print follows this pipeline [S1][S3]:

```
[CAD / download] → model file (STL/OBJ/3MF/STEP)
                           ↓
                    [Slicer software]
                           ↓
              toolpath / G-code (.gcode, .ufp, .form …)
                           ↓
                    [Printer executes]
```

The printer never directly reads a model file — the slicer is the mandatory bridge [S4].

---

### Model Files (pre-slicing)

These describe the 3D shape and are the files you create, download, or export from CAD.

#### STL (.stl) — the universal standard [S1][S2]
- Represents surface geometry as a mesh of triangular facets.
- ASCII or binary encoding; no color, no texture, no material data.
- Supported by every slicer and virtually every CAD tool.
- Best for: functional/structural parts, maximum compatibility, beginners.
- Limitation: cannot encode material gradients; scale is ambiguous (metric vs. imperial).

#### OBJ (.obj) — texture and color support [S2][S5]
- Supports UV-mapped textures and color via companion `.mtl` file and image files.
- Uses polygons (not strictly triangles) for better curved-surface approximation.
- Common for artistic models, 3D scans, figurines, game assets.
- Caveat: moving the file without its companion `.mtl` and image files loses all textures.

#### 3MF (.3mf) — the modern standard [S1][S2][S6]
- Developed in 2015 by the 3MF Consortium (Microsoft, Autodesk, 3D Systems, Stratasys, HP).
- XML-based package: stores geometry, full-color textures, multi-material data, embedded slicer settings, authorship metadata.
- Better compression than STL; eliminates scale ambiguity.
- Reduces non-manifold mesh errors.
- Rapidly becoming the preferred standard; legacy software/older printers may not fully support it yet.
- Best for: multi-material prints, professional workflows, sharing setups that should be reproducible.

#### AMF (.amf) — capable but obsolete in practice [S2]
- XML-based additive-manufacturing format; supports material gradients and color.
- Never achieved wide adoption; largely superseded by 3MF.

#### STEP (.stp / .step) — engineering-grade CAD exchange [S7][S8]
- Full B-Rep (boundary representation) solid model; retains parametric, assembly, and curved-surface data.
- Most reliable format for exchanging between professional CAD systems (SolidWorks, Fusion 360, CATIA, NX).
- Converting STEP → STL/3MF produces cleaner mesh results than converting from IGES.
- Some modern slicers (Formlabs PreForm 3.43+, Dec 2024) now import STEP directly [S9].
- Best for: engineering parts with tight tolerances, manufacturing workflows.

#### IGES (.iges / .igs) — legacy CAD exchange [S7][S8]
- Oldest neutral CAD exchange format (ANSI standard, first published 1980; last version 1996).
- Primarily surface-model focused; more prone to translation errors than STEP.
- Being actively superseded by STEP; still widely compatible as a fallback.

### Toolpath / Machine Files (post-slicing)

These are what the printer's controller actually reads and executes.

#### G-code (.gcode, .gco, .gc) — the universal machine language [S3][S4][S10]
- A CNC-derived instruction set for controlling motion, temperature, extrusion, and fan speed.
- **G-commands**: motion — `G1` (linear move), `G28` (home all axes), `G90` (absolute positioning).
- **M-commands**: machine functions — `M104` (set extruder temperature), `M106` (fan on), `M107` (fan off).
- A single print can contain tens of thousands of commands.
- Generated by slicer software: Cura, PrusaSlicer, Bambu Studio, Simplify3D, Creality Print.
- Used primarily for FDM/FFF printers; some SLA/DLP machines also accept G-code variants.

#### Proprietary toolpath formats
- **.ufp** (Ultimaker File Package): Cura packages G-code + thumbnail + printer metadata into one file for Ultimaker printers [S11].
- **.form**: Formlabs PreForm proprietary format for SLA/SLS printers (Form 3, Form 4, Fuse series) [S9].
- **.xyz**: DaVinci Color printers combine movement commands with ink-head color commands [S10].
- **.gx** (FlashForge): similar packaged format.
- Stratasys printers use GrabCAD Print software, which accepts many input formats and produces proprietary job files.

### Slicing Software

The slicer is the required intermediate step — it takes your model file and produces machine-ready instructions [S4]:

| Slicer | Common Use |
|--------|-----------|
| Cura (Ultimaker) | FDM — very popular, free, wide printer support |
| PrusaSlicer | FDM — open source, high accuracy (best surface accuracy per medical study) |
| Bambu Studio | FDM — optimized for Bambu printers |
| Simplify3D | FDM — commercial, fine-grained control |
| Chitubox / Lychee | Resin (SLA/DLP/MSLA) |
| Formlabs PreForm | Formlabs SLA/SLS — free, proprietary |

### Differences by Printer Technology

| Technology | Common Model Format Input | Machine File Output |
|------------|--------------------------|---------------------|
| FDM (consumer/hobbyist) | STL, 3MF, OBJ | G-code (.gcode) |
| SLA/DLP/MSLA (resin) | STL, OBJ, 3MF | G-code variant or proprietary (.form, .ctb) |
| SLS (powder) | STL, 3MF | Proprietary job file (EOS, Sinterit) |
| DMLS/SLM (metal) | STL, STEP | Proprietary machine file |
| PolyJet (Stratasys) | STL, OBJ, STEP, native CAD via GrabCAD | Proprietary .job |

## Constraints

- All printers ultimately need either G-code or a vendor-proprietary equivalent — the model file is never sufficient on its own.
- For multi-color/multi-material prints, STL must be replaced with 3MF or OBJ.
- Legacy printers (pre-2018 era) may not accept 3MF; STL is the safest universal fallback.
- SLA/DLP printers require their own slicers (Chitubox, Lychee, PreForm) — standard FDM slicers cannot produce resin toolpaths.

## Solution Comparison

| Format | Geometry | Color/Texture | Slicer Support | Best Use |
|--------|----------|---------------|----------------|---------|
| STL | Triangular mesh | None | Universal | Simple functional parts |
| OBJ | Polygon mesh | Yes (via .mtl) | Good | Artistic/scanned models |
| 3MF | Mesh + metadata | Yes (built-in) | Modern slicers | Multi-material, pro workflows |
| AMF | Mesh + material | Yes | Limited | Legacy only |
| STEP | B-Rep solid | Material data | Engineering slicers | Precision engineering parts |
| IGES | Surface model | Limited | Wide but aging | Legacy CAD exchange |
| G-code | N/A (instructions) | N/A | Printer firmware | Final printer execution |

## Recommendation

For most 3D printing workflows:

1. **Use STL** to get started — maximum compatibility with every slicer and printer.
2. **Upgrade to 3MF** when: sharing print setups, working with multi-material, or using modern printers (Bambu, Prusa, Ultimaker) that fully support it.
3. **Use STEP** when: you have a precision engineering part and your slicer (e.g., Formlabs PreForm 3.43+) or manufacturing service accepts it directly — it will produce cleaner meshes than exporting to STL first.
4. **Always slice before printing** — the output is G-code (or a proprietary equivalent) regardless of input format.

**Implementation outline for anyone wanting to print a 3D design:**
1. Create or download a model (preferably STL or 3MF).
2. Open it in a slicer matching your printer (Cura for most FDM; PreForm for Formlabs; Chitubox for resin).
3. Configure print settings (layer height, infill, supports, temperatures).
4. Export/slice → G-code (or printer-specific format).
5. Transfer to printer via SD card, USB, network, or cloud.
6. Print.

**Risks and mitigations:**
- STL scale ambiguity → use 3MF or verify units on import in your slicer.
- Missing texture files with OBJ → always keep .mtl and image files together, or prefer 3MF for color work.
- Slicer / printer firmware mismatch → match the slicer profile to your printer model.
